# Heading

`Heading` is a semantic wrapper around the `Text` component, used to render heading tags (`<h1>` to `<h6>`) with consistent typography and scale using Tailwind CSS. It ensures visual hierarchy and accessibility while allowing full styling flexibility.

## Features

- Supports all HTML heading levels (`h1`–`h6`) via `element` prop
- Predefined visual sizes mapped to semantic levels
- Composable with Tailwind utilities via `className`
- Built on top of `Text` for consistent theming
- Fully accessible and SEO-friendly

## Usage Guidelines

- Use `Heading` to define clear sections and page hierarchy.
- Match `element` (e.g., `h2`) to the logical structure of the document.
- Use Tailwind utilities in `className` for custom spacing, font, and color.
- Combine with layout components like `Box`, `Section`, or `Stack`.

## Do

- Use `h1` for main titles, `h2` for subsections, and follow a logical order
- Add utility classes (`mt-*`, `text-*`) using `className` as needed
- Use inside layout wrappers for clean structure and alignment

## Don’t

- Use `Heading` just for visual text that’s not a section heading
- Rely only on font size for hierarchy — semantic structure matters
- Over-style headings inline; prefer reusable class tokens
