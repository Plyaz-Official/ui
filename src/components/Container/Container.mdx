# Container

`Container` is a responsive layout wrapper that centers content horizontally and constrains its width for better readability. It is typically used to wrap high-level sections like headers, footers, or main content.

## Features

- Centers content using `mx-auto`
- Responsive padding: `px-4 sm:px-6 lg:px-8`
- Default max width via `max-w-7xl` (customizable)
- Inherits all native HTML props (`id`, `role`, `aria-*`, `data-*`, etc.)

## Usage Guidelines

- Use `Container` to wrap major layout regions.
- Combine with layout components like `Box`, `Grid`, or `Flex`.
- Customize spacing and max width via `className`.
- Use semantic tags through composition if needed (`<main>`, `<section>`, etc.)

## Do

- Wrap key layout sections to maintain alignment and consistency
- Extend styling using Tailwind utility classes via `className`
- Pass accessibility attributes via props (`aria-label`, `role`, etc.)

## Don’t

- Nest containers unnecessarily — one per layout section is usually sufficient
- Apply `max-w-*` globally across the app
- Ignore spacing on small breakpoints — test with Storybook’s viewport tool
