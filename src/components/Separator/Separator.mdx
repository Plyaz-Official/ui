# Separator

The `Separator` component provides a visual separator between content sections. It is built with Radix UI and styled with Tailwind CSS, offering both horizontal and vertical orientations.

## Features

- **Visual Separation**: Provides clear visual separation between content sections
- **Accessibility**: Built with Radix UI primitives for full accessibility support
- **Orientation Support**: Supports both horizontal and vertical orientations
- **Customizable**: Supports custom styling and theming
- **Decorative**: Can be marked as decorative for screen readers
- **Responsive**: Adapts to different screen sizes and orientations

## Props

| Prop          | Type                           | Default        | Description                                                           |
| ------------- | ------------------------------ | -------------- | --------------------------------------------------------------------- |
| `orientation` | `"horizontal" \| "vertical"`   | `"horizontal"` | The orientation of the separator                                      |
| `decorative`  | `boolean`                      | `true`         | Whether the separator is decorative (not announced by screen readers) |
| `className`   | `string`                       | -              | Additional CSS classes to apply to the separator                      |
| `...props`    | `SeparatorPrimitive.RootProps` | -              | Additional props passed to the Radix UI Separator root                |

## Styling

The `Separator` component uses Tailwind CSS classes and can be customized with additional classes:

```tsx
<Separator className='h-2 bg-primary rounded-full' />
```

## Accessibility

The `Separator` component is built with Radix UI primitives and provides:

- **Screen Reader Support**: Proper ARIA attributes and roles
- **Decorative Support**: Can be marked as decorative for screen readers
- **Focus Management**: Focus indicators and proper focus handling
- **Keyboard Navigation**: Full keyboard support for navigation

## Performance

The `Separator` component is optimized for performance:

- **Lightweight**: Minimal DOM footprint
- **Efficient Rendering**: Optimized for smooth rendering
- **Memory Efficient**: Minimal memory usage
- **Responsive**: Adapts to different screen sizes and orientations

## Best Practices

1. **Use appropriate orientation**: Choose horizontal or vertical based on layout
2. **Provide meaningful content**: Ensure content is accessible and meaningful
3. **Handle decorative state**: Mark as decorative when appropriate
4. **Test with screen readers**: Ensure accessibility works properly
5. **Consider mobile**: Test on mobile devices for touch interactions
