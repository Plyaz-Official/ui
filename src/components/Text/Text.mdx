# Text

The `Text` component is a flexible and semantic typography utility used to render elements like paragraphs, captions, or inline spans with consistent Tailwind-based styles.

## Features

- Renders semantic HTML tags like `p`, `span`, or heading levels (`h1`–`h6`)
- Supports variants and sizes aligned with the design system
- Applies consistent text styles via Tailwind CSS
- Inherits native HTML props (e.g., `id`, `className`, `aria-*`, `data-*`)

## Usage Guidelines

- Use `Text` instead of raw HTML tags to ensure consistency
- Apply the correct semantic tag (`h1`, `p`, `span`, etc.) based on content
- Use `size` and `variant` props to control visual appearance
- Combine with layout primitives like `Box`, `Stack`, or `Container` for structured design

## Do

- Use semantic tags to match content structure and improve accessibility
- Style using Tailwind utilities and design system tokens
- Pair with layout components for responsive designs
- Use consistent variants (e.g. `caption`, `body`, `heading`) across your app

## Don’t

- Use raw HTML elements (`<p>`, `<h1>`) without design consistency
- Inline hardcoded styles—prefer Tailwind classes or props
- Skip heading levels (e.g., jumping from `h2` to `h4`)
- Use heading tags for non-heading content
