# Paragraph

The `Paragraph` component is a semantic and styled wrapper around the native `<p>` tag. It ensures consistent typography for body text using Tailwind utility classes while supporting accessibility and readability best practices.

## Features

- Semantically renders a `<p>` tag
- Applies consistent body text styling via Tailwind
- Supports size variants (`xs`, `sm`, `base`, `lg`)
- Inherits all native HTML props (`className`, `id`, `aria-*`, `data-*`, etc.)

## Usage Guidelines

- Use `Paragraph` for descriptive content, long-form text, or any standard body copy.
- Select the appropriate `size` based on visual hierarchy and context.
- Use `className` for spacing, alignment, or color overrides via Tailwind.

## Do

- Use `Paragraph` instead of raw `<p>` to enforce consistent styling
- Match size variants to content type and screen size
- Compose with layout components like `Box`, `Stack`, or `Container`

## Don’t

- Nest block-level elements like headings or lists inside a `Paragraph`
- Mix inconsistent text sizes in the same section
- Overuse inline styles—prefer Tailwind utilities or design tokens
