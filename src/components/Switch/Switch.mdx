# Switch

A control that allows the user to toggle between checked and not checked.

## Features

- **Toggle Control**: Provides a toggle switch for boolean values
- **Accessibility**: Built with Radix UI primitives for full accessibility support
- **Customizable**: Supports custom styling and theming
- **Keyboard Navigation**: Full keyboard support for navigation
- **Touch Support**: Optimized for touch devices

## Styling

The `Switch` component uses Tailwind CSS classes and can be customized with additional classes:

```tsx
<Switch className='data-[state=checked]:bg-primary [&>[data-slot=switch-thumb]]:size-5' />
```

## Accessibility

The `Switch` component is built with Radix UI primitives and provides:

- **Keyboard Navigation**: Full keyboard support for navigation
- **Screen Reader Support**: Proper ARIA attributes and roles
- **Focus Management**: Focus indicators and proper focus handling
- **Touch Support**: Optimized for touch devices
- **State Announcements**: Announces state changes to screen readers

## Best Practices

1. **Use appropriate labels**: Always provide clear labels for switches
2. **Handle state changes**: Use `onCheckedChange` for controlled state
3. **Test with keyboard**: Ensure keyboard navigation works properly
4. **Use controlled state**: Use controlled state for complex interactions
5. **Provide visual feedback**: Show clear visual feedback for state changes
