# Switch

The `Switch` component provides a toggle switch control for boolean values. It is built with Radix UI and styled with Tailwind CSS, offering a flexible and accessible switch solution.

## Features

- **Toggle Control**: Provides a toggle switch for boolean values
- **Accessibility**: Built with Radix UI primitives for full accessibility support
- **Customizable**: Supports custom styling and theming
- **Responsive**: Adapts to different screen sizes and orientations
- **Keyboard Navigation**: Full keyboard support for navigation
- **Touch Support**: Optimized for touch devices

import { Switch } from '@/components';
import { useState } from 'react';

function InteractiveSwitch() {
const [checked, setChecked] = useState(false);
const [clickCount, setClickCount] = useState(0);

const handleCheckedChange = (newChecked: boolean) => {
setChecked(newChecked);
setClickCount(prev => prev + 1);
};

return (
<div className="space-y-4">
<div className="flex items-center space-x-2">
<Switch
          checked={checked}
          onCheckedChange={handleCheckedChange}
        />
<label className="text-sm font-medium">
Interactive Switch
</label>
</div>
<div className="text-sm text-muted-foreground">
Current state: {checked ? 'On' : 'Off'}
</div>
<div className="text-sm text-muted-foreground">
Toggle count: {clickCount}
</div>
</div>
);
}
`}
/>

## Props

| Prop              | Type                         | Default | Description                                         |
| ----------------- | ---------------------------- | ------- | --------------------------------------------------- |
| `checked`         | `boolean`                    | -       | The checked state of the switch                     |
| `defaultChecked`  | `boolean`                    | -       | The default checked state of the switch             |
| `disabled`        | `boolean`                    | `false` | Whether the switch is disabled                      |
| `required`        | `boolean`                    | `false` | Whether the switch is required                      |
| `name`            | `string`                     | -       | The name of the switch                              |
| `value`           | `string`                     | -       | The value of the switch                             |
| `id`              | `string`                     | -       | The id of the switch                                |
| `className`       | `string`                     | -       | Additional CSS classes to apply to the switch       |
| `onCheckedChange` | `(checked: boolean) => void` | -       | Called when the checked state changes               |
| `...props`        | `SwitchPrimitive.RootProps`  | -       | Additional props passed to the Radix UI Switch root |

## Styling

The `Switch` component uses Tailwind CSS classes and can be customized with additional classes:

```tsx
<Switch className='data-[state=checked]:bg-primary [&>[data-slot=switch-thumb]]:size-5' />
```

## Accessibility

The `Switch` component is built with Radix UI primitives and provides:

- **Keyboard Navigation**: Full keyboard support for navigation
- **Screen Reader Support**: Proper ARIA attributes and roles
- **Focus Management**: Focus indicators and proper focus handling
- **Touch Support**: Optimized for touch devices
- **State Announcements**: Announces state changes to screen readers

## Performance

The `Switch` component is optimized for performance:

- **Efficient Rendering**: Only re-renders when necessary
- **Smooth Animations**: Provides smooth state transitions
- **Memory Efficient**: Minimal memory usage
- **Responsive**: Adapts to different screen sizes and orientations

## Best Practices

1. **Use appropriate labels**: Always provide clear labels for switches
2. **Handle state changes**: Use `onCheckedChange` for controlled state
3. **Test with keyboard**: Ensure keyboard navigation works properly
4. **Consider mobile**: Test on mobile devices for touch interactions
5. **Use controlled state**: Use controlled state for complex interactions
6. **Provide visual feedback**: Show clear visual feedback for state changes
7. **Group related switches**: Group related switches together logically
