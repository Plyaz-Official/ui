# Tooltip

The `Tooltip` component provides contextual information when users hover over or focus on an element. It is built with Radix UI and styled with Tailwind CSS, offering a flexible and accessible tooltip solution.

## Features

- **Contextual Information**: Provides helpful information on hover or focus
- **Accessibility**: Built with Radix UI primitives for full accessibility support
- **Customizable**: Supports custom styling and theming
- **Responsive**: Adapts to different screen sizes and positions
- **Keyboard Navigation**: Full keyboard support for navigation
- **Touch Support**: Optimized for touch devices

## Props

| Component         | Prop                      | Type                                      | Default    | Description                                                   |
| ----------------- | ------------------------- | ----------------------------------------- | ---------- | ------------------------------------------------------------- |
| `TooltipProvider` | `delayDuration`           | `number`                                  | `0`        | The delay duration in milliseconds before showing the tooltip |
| `TooltipProvider` | `skipDelayDuration`       | `number`                                  | `0`        | The delay duration in milliseconds before hiding the tooltip  |
| `TooltipProvider` | `disableHoverableContent` | `boolean`                                 | `false`    | Whether to disable hoverable content                          |
| `Tooltip`         | `open`                    | `boolean`                                 | -          | Whether the tooltip is open                                   |
| `Tooltip`         | `defaultOpen`             | `boolean`                                 | `false`    | Whether the tooltip is open by default                        |
| `Tooltip`         | `onOpenChange`            | `(open: boolean) => void`                 | -          | Called when the open state changes                            |
| `TooltipContent`  | `side`                    | `"top" \| "right" \| "bottom" \| "left"`  | `"top"`    | The side of the tooltip                                       |
| `TooltipContent`  | `align`                   | `"start" \| "center" \| "end"`            | `"center"` | The alignment of the tooltip                                  |
| `TooltipContent`  | `sideOffset`              | `number`                                  | `0`        | The offset from the side                                      |
| `TooltipContent`  | `alignOffset`             | `number`                                  | `0`        | The offset from the alignment                                 |
| `TooltipContent`  | `collisionBoundary`       | `Element \| null`                         | -          | The collision boundary element                                |
| `TooltipContent`  | `collisionPadding`        | `number \| Partial<Record<Side, number>>` | `0`        | The collision padding                                         |
| `TooltipContent`  | `arrowPadding`            | `number`                                  | `0`        | The padding between the arrow and the edge                    |
| `TooltipContent`  | `sticky`                  | `"partial" \| "always"`                   | -          | The sticky behavior of the tooltip                            |
| `TooltipContent`  | `hideWhenDetached`        | `boolean`                                 | `false`    | Whether to hide the tooltip when detached                     |
| `TooltipContent`  | `className`               | `string`                                  | -          | Additional CSS classes to apply to the tooltip                |
| `...props`        | -                         | `TooltipPrimitive.RootProps`              | -          | Additional props passed to the Radix UI Tooltip root          |

## Styling

The `Tooltip` component uses Tailwind CSS classes and can be customized with additional classes:

```tsx
<TooltipContent className='bg-blue-500 text-white' />
<TooltipContent side='right' className='ml-2' />
<TooltipContent sideOffset={10} className='mt-2' />
```

## Accessibility

The `Tooltip` component is built with Radix UI primitives and provides:

- **Keyboard Navigation**: Full keyboard support for navigation
- **Screen Reader Support**: Proper ARIA attributes and roles
- **Focus Management**: Focus indicators and proper focus handling
- **Touch Support**: Optimized for touch devices
- **State Announcements**: Announces state changes to screen readers

## Performance

The `Tooltip` component is optimized for performance:

- **Efficient Rendering**: Only re-renders when necessary
- **Smooth Animations**: Provides smooth state transitions
- **Memory Efficient**: Minimal memory usage
- **Responsive**: Adapts to different screen sizes and positions

## Best Practices

1. **Use appropriate content**: Keep tooltip content concise and helpful
2. **Handle state changes**: Use `onOpenChange` for controlled state
3. **Test with keyboard**: Ensure keyboard navigation works properly
4. **Consider mobile**: Test on mobile devices for touch interactions
5. **Use controlled state**: Use controlled state for complex interactions
6. **Provide visual feedback**: Show clear visual feedback for tooltip state
7. **Group related tooltips**: Group related tooltips logically
8. **Use consistent positioning**: Use consistent positioning within groups
