# Input

`Input` is a foundational form element used for capturing user text input. It supports all native HTML input attributes and is styled using Tailwind CSS utilities for consistent UI across your application.

The ShadCN `Input` is built on top of the native `<input />` element with extended styling, theming, and accessibility support.

## Features

- **Complete HTML5 Input Support**: All native `<input>` types including text, email, password, number, search, url, tel, date, time, and file
- **Accessibility First**: Built-in ARIA support, focus management, and screen reader compatibility
- **Validation Ready**: Supports `required`, `pattern`, `min`, `max`, `maxLength`, and other validation attributes
- **State Management**: Handles disabled, readonly, error states with proper visual feedback
- **Tailwind Integration**: Styled using Tailwind CSS with consistent theming
- **Form Library Compatible**: Works seamlessly with react-hook-form, Formik, and other form libraries
- **Dark Mode Support**: Automatic dark mode styling
- **Responsive Design**: Adapts to different screen sizes

## Usage Guidelines

- Always associate an `Input` with a `Label` for better accessibility.
- Use `type="email"`, `type="password"`, etc., to enforce input types.
- Combine with `FormField` and `FormItem` to manage validation.
- Avoid using inline stylesâ€”prefer Tailwind utilities or class extensions.

## Do

- Use appropriate `type` attributes for better UX (email, password, number, etc.)
- Provide meaningful `placeholder` text for guidance
- Use `required` for mandatory fields
- Implement proper validation with error messages
- Wrap inside `FormField` and `FormItem` when using with form libraries
- Use `aria-invalid` for error states
- Set `maxLength` for text inputs with character limits
- Use `min`/`max` for number inputs with constraints

## Don't

- Use the Input without proper labeling
- Handle validation purely with visual cues without accessible messages
- Mix input types that conflict with expected user behavior
- Use placeholder text as the only label
- Forget to handle loading/disabled states appropriately
