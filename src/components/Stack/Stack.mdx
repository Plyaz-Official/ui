# Stack

`Stack` is a layout utility component that arranges its children in a horizontal or vertical direction using Tailwind’s `space-x-*` or `space-y-*` utilities. It's ideal for quick, clean spacing without custom margins or complex layout structures.

---

## Features

- Directional layout: `horizontal` or `vertical`
- Applies consistent spacing using `space-x-*` / `space-y-*`
- Composable with other layout components (`Box`, `Card`, `Button`)
- Supports all native HTML props (`id`, `aria-*`, `data-*`, etc.)
- Clean abstraction over Flexbox for simpler use cases

## Usage Guidelines

- Use `Stack` to evenly space elements in one direction.
- Prefer `horizontal` for row-based UI (buttons, tags) and `vertical` for columns (forms, cards).
- Pass spacing via props or Tailwind utilities in `className`.
- Use inside layout primitives like `Container`, `Box`, or `Section`.

## Do

- Use direction prop ("horizontal" or "vertical") to switch layout easily
- Nest reusable UI elements like Button, Card, Text, or custom components inside Stack
- Leverage the element prop to render semantic tags (section, nav, etc.) as needed

## Don’t

- Use Stack for advanced layouts requiring wrapping, breakpoints, or grid behavior — use Grid or custom Flex instead
- Add complex layout logic inside Stack — keep it focused on spacing and direction only
- Override flex-\* direction manually via className, which may break layout intent
