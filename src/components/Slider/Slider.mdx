# Slider

An input where the user selects a value from within a given range.

## Features

- **Range Input**: Provides a visual range input control for selecting values
- **Accessibility**: Built with Radix UI primitives for full accessibility support
- **Flexible Values**: Supports single values, ranges, and multiple values
- **Customizable**: Supports custom styling and theming
- **Responsive**: Adapts to different screen sizes and orientations
- **Keyboard Navigation**: Full keyboard support for navigation
- **Touch Support**: Optimized for touch devices

## Styling

The `Slider` component uses Tailwind CSS classes and can be customized with additional classes:

```tsx
<Slider
  defaultValue={[50]}
  max={100}
  step={1}
  className='[&_[data-slot=slider-range]]:bg-primary [&_[data-slot=slider-thumb]]:size-6'
/>
```

## Accessibility

The `Slider` component is built with Radix UI primitives and provides:

- **Keyboard Navigation**: Full keyboard support for navigation
- **Screen Reader Support**: Proper ARIA attributes and roles
- **Focus Management**: Focus indicators and proper focus handling
- **Touch Support**: Optimized for touch devices
- **Value Announcements**: Announces value changes to screen readers

## Best Practices

1. **Use appropriate step values**: Choose step values that make sense for your use case
2. **Provide meaningful labels**: Use `aria-label` for screen readers
3. **Handle value changes**: Use `onValueChange` for real-time updates
4. **Test with keyboard**: Ensure keyboard navigation works properly
5. **Consider mobile**: Test on mobile devices for touch interactions
6. **Use controlled state**: Use controlled state for complex interactions
7. **Provide visual feedback**: Show current values and ranges clearly
