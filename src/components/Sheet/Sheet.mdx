# Sheet

The `Sheet` component provides a slide-out panel that can be triggered from any side of the screen. It is built with Radix UI and styled with Tailwind CSS, offering a flexible and accessible overlay solution.

## Features

- **Slide-out Panels**: Provides slide-out panels from any side of the screen
- **Accessibility**: Built with Radix UI primitives for full accessibility support
- **Flexible Positioning**: Supports top, right, bottom, and left positioning
- **Customizable**: Supports custom styling and theming
- **Responsive**: Adapts to different screen sizes and orientations
- **Keyboard Navigation**: Full keyboard support for navigation
- **Touch Support**: Optimized for touch devices

## Props

### Sheet

| Prop           | Type                       | Default | Description                                        |
| -------------- | -------------------------- | ------- | -------------------------------------------------- |
| `children`     | `ReactNode`                | -       | The content to be displayed inside the sheet       |
| `open`         | `boolean`                  | -       | Whether the sheet is open                          |
| `onOpenChange` | `(open: boolean) => void`  | -       | Called when the open state changes                 |
| `...props`     | `SheetPrimitive.RootProps` | -       | Additional props passed to the Radix UI Sheet root |

### SheetTrigger

| Prop       | Type                          | Default | Description                                           |
| ---------- | ----------------------------- | ------- | ----------------------------------------------------- |
| `asChild`  | `boolean`                     | `false` | Whether to merge props with child element             |
| `children` | `ReactNode`                   | -       | The content to be displayed inside the trigger        |
| `...props` | `SheetPrimitive.TriggerProps` | -       | Additional props passed to the Radix UI Sheet trigger |

### SheetClose

| Prop       | Type                        | Default | Description                                         |
| ---------- | --------------------------- | ------- | --------------------------------------------------- |
| `asChild`  | `boolean`                   | `false` | Whether to merge props with child element           |
| `children` | `ReactNode`                 | -       | The content to be displayed inside the close button |
| `...props` | `SheetPrimitive.CloseProps` | -       | Additional props passed to the Radix UI Sheet close |

### SheetContent

| Prop        | Type                                     | Default   | Description                                           |
| ----------- | ---------------------------------------- | --------- | ----------------------------------------------------- |
| `side`      | `"top" \| "right" \| "bottom" \| "left"` | `"right"` | The side from which the sheet slides out              |
| `children`  | `ReactNode`                              | -         | The content to be displayed inside the sheet          |
| `className` | `string`                                 | -         | Additional CSS classes to apply to the sheet          |
| `...props`  | `SheetPrimitive.ContentProps`            | -         | Additional props passed to the Radix UI Sheet content |

### SheetHeader

| Prop        | Type                                   | Default | Description                                   |
| ----------- | -------------------------------------- | ------- | --------------------------------------------- |
| `children`  | `ReactNode`                            | -       | The content to be displayed inside the header |
| `className` | `string`                               | -       | Additional CSS classes to apply to the header |
| `...props`  | `React.HTMLAttributes<HTMLDivElement>` | -       | Additional props passed to the header div     |

### SheetFooter

| Prop        | Type                                   | Default | Description                                   |
| ----------- | -------------------------------------- | ------- | --------------------------------------------- |
| `children`  | `ReactNode`                            | -       | The content to be displayed inside the footer |
| `className` | `string`                               | -       | Additional CSS classes to apply to the footer |
| `...props`  | `React.HTMLAttributes<HTMLDivElement>` | -       | Additional props passed to the footer div     |

### SheetTitle

| Prop        | Type                        | Default | Description                                         |
| ----------- | --------------------------- | ------- | --------------------------------------------------- |
| `children`  | `ReactNode`                 | -       | The content to be displayed inside the title        |
| `className` | `string`                    | -       | Additional CSS classes to apply to the title        |
| `...props`  | `SheetPrimitive.TitleProps` | -       | Additional props passed to the Radix UI Sheet title |

### SheetDescription

| Prop        | Type                              | Default | Description                                               |
| ----------- | --------------------------------- | ------- | --------------------------------------------------------- |
| `children`  | `ReactNode`                       | -       | The content to be displayed inside the description        |
| `className` | `string`                          | -       | Additional CSS classes to apply to the description        |
| `...props`  | `SheetPrimitive.DescriptionProps` | -       | Additional props passed to the Radix UI Sheet description |


## Accessibility

The `Sheet` component is built with Radix UI primitives and provides:

- **Keyboard Navigation**: Full keyboard support for navigation
- **Screen Reader Support**: Proper ARIA attributes and roles
- **Focus Management**: Focus indicators and proper focus handling
- **Touch Support**: Optimized for touch devices
- **Escape Key**: Closes the sheet when pressed

## Performance

The `Sheet` component is optimized for performance:

- **Lazy Loading**: Content is only rendered when the sheet is open
- **Smooth Animations**: Provides smooth slide-in/out animations
- **Memory Efficient**: Only renders visible content
- **Responsive**: Adapts to different screen sizes and orientations

## Best Practices

1. **Use appropriate side**: Choose the side based on your layout and user experience
2. **Provide meaningful content**: Ensure content is accessible and meaningful
3. **Handle loading states**: Show loading indicators for dynamic content
4. **Test with keyboard**: Ensure keyboard navigation works properly
5. **Consider mobile**: Test on mobile devices for touch interactions
6. **Use controlled state**: Use controlled state for complex interactions
7. **Provide close options**: Always provide a way to close the sheet
